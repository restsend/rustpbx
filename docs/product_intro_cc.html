场景一：呼入全链路

客户来电→ IVR 导航→ DTMF 选择→ 技能组匹配→ 入队→ ACD 分配→ 坐席接听→ CSAT 评分

ACD 分配策略

策略逻辑
LongestIdle最长空闲优先
LeastAnswered接听最少优先
RoundRobin轮询分配
SkillBased技能+空闲+优先级打分
Random随机分配

溢出降级链

  1. 同技能组重试
  2. 溢出到备用技能组
  3. 回到 IVR 二次路由
  4. 转语音信箱
  5. 创建回呼任务
  6. 播放提示后挂断

坐席状态机

Offline → Idle → Ringing
              ↓        ↓
            Away    Busy
                     ↓
                  WrapUp → Idle

WrapUp 话后整理，按技能组配置时长
Screen Pop 接听时自动弹屏 CRM

CSAT 评分流程

  1. 挂断后判断: 接通坐席 + 通话 ≥ 20s + 去重(4h内1次)
  2. 转入评分 IVR → 播放提示音
  3. DTMF 按键 1-5 分
  4. 落 survey_result (trace_id/agent/csat/talk_secs)
  5. 播感谢语 → 挂断

Listen 监听

Provider 接口协议

每步 POST 携带完整上下文，Provider 返回 ActionNode:

// 请求 POST {url}
{
  "session_id": "call_abc123",
  "caller": "1001", "callee": "2000",
  "variables": { "phone": "138..." },
  "event": { "type": "dtmf", "digit": "1" }
}

// 响应
{
  "type": "dtmf_menu",
  "greeting": "menu.wav",
  "entries": {
    "1": { "type": "transfer", "target": "2001" },
    "2": { "type": "queue", "target": "support" }
  }
}

IVR Editor 工具链

  • 可视化配置 — 填写 Provider URL / Headers / 重试策略
  • 一键发布 — 生成配置文件，路由层自动注入 StepIvrApp
  • 实时 Trace — 每步 action_type/耗时/错误 面板展示
  • 诊断页测试 — Web Dialer + DTMF 键盘 + 2s 轮询 Trace
  • RWI 推送 — 步模式 IVR 实时推送 ivr_step_trace 事件
  • 3次重试 — 全部失败播 error.wav → 挂断

场景五：集群模式

Call Proxy

PSTN 接入 + 媒体锚定
ConferenceManager 会议管理
所有 leg 媒体汇聚

CC Proxy

坐席侧接入 (WS/SIP-WS)
有状态 (BC密语持有双Session)
originate_call → Call Proxy

全网格事件同步

通过 SIP MESSAGE 同步四类事件:

Locator注册信息跨节点同步
Presence坐席状态一致性
AgentStatus完整坐席快照(技能/优先级/instance_id)
QueueEvent跨节点队列可见+原子抢占

Home Proxy 路由

每个注册记录 home_proxy
本节点 → 直接路由
远程节点 → SIP 转发 INVITE
坐席跨节点迁移自动跟随

集群安全

  • Peer IP 豁免 SIP Auth (同域)
  • Peer IP 豁免 AMI 端点
  • 集群 MESSAGE IP 白名单验证
  • 自消息回环检测+丢弃
  • 跨节点队列 claimed_by 原子抢占

集群管理

端点功能
POST /cluster/ping探测 peer 健康 (延迟/uptime/呼叫数)
GET /cluster/reload_configSSE 串行重载所有节点
POST /cluster/dispatch_command向 peer 转发呼叫控制指令
GET /cluster/list_calls汇总所有节点活跃通话
